## 1. Зміни в Базі Даних (SQL)

### А. Емуляція зовнішніх ключів (Foreign Keys) через Тригери
Додано таблицю `maintenance_logs` (Журнал обслуговування), яка має зв'язок 1:M зі станціями.
* **Особливість:** Фізичний `CONSTRAINT FOREIGN KEY` відсутній.
* **Реалізація:** Цілісність даних забезпечується тригерами:
    * `trg_maintenance_insert_check`: Блокує вставку запису, якщо `station_id` не існує.
    * `trg_station_delete_check`: Блокує видалення станції, якщо у неї є записи в журналі.

### Б. Збережені процедури та Курсори
Реалізовано набір процедур для складної бізнес-логіки:
1.  **Параметризована вставка (`sp_insert_panel_type`):** Безпечне додавання типів панелей.
2.  **M:M за значеннями (`sp_link_user_to_station`):** Створення зв'язку між юзером та станцією, використовуючи їх email та назву (пошук ID всередині процедури).
3.  **Пакетна вставка (`sp_insert_nonames`):** Генерація 10 тестових користувачів у циклі.
4.  **Динамічна агрегація (`sp_dynamic_aggregation`):** Універсальна процедура, яка приймає назву таблиці, стовпця та функцію (AVG, MAX, MIN) і повертає результат, використовуючи Dynamic SQL (`PREPARE`/`EXECUTE`).
5.  **Робота з курсорами (`sp_cursor_create_tables`):** Динамічне створення нових таблиць. Курсор проходить по всіх типах панелей і створює для кожного типу окрему таблицю з іменем у форматі `Type_Timestamp`.

### В. Тригери (Бізнес-правила)
1.  **Валідація даних (`trg_user_surname_check`):** Забороняє реєстрацію користувачів, прізвище яких закінчується на "00".
2.  **Аудит (`trg_log_station_delete`):** При успішному видаленні станції автоматично записує ID та час видалення у спеціальну таблицю `audit_log_deleted`.
3.  **Формат вводу (`trg_address_format_check`):** Перевіряє формат номера будинку.

## 2. Інтеграція з Backend (Flask)

Створено окремий шар для роботи зі збереженими процедурами, щоб не порушувати існуючу структуру CRUD:
* **Controller:** `app/user/controller/stored_program_controller.py` — нові ендпоінти `/api/proc/...`.
* **Service:** `app/user/service/stored_program_service.py` — бізнес-логіка виклику процедур.
* **DAO:** `app/user/dao/stored_program_dao.py` — виконання "сирих" SQL запитів (`CALL ...`) через SQLAlchemy session.
